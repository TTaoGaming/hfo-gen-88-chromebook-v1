# HFO Daily Card — Format Options (Review)

Purpose: pick a 1‑page, low‑cognitive‑load format that adds per‑section guidance + inline examples.
Scope: [Obsidian2025/03-resources/hfo_cards/hfo_daily_card.md](Obsidian2025/03-resources/hfo_cards/hfo_daily_card.md)

## Option A — Micro‑Guidance Line
**Format:** Each section gets one line: Goal • Rule • Example. Inline examples remain on the fields.
**Why:** Minimal extra text; fastest scan; preserves 1‑page feel.
**Cognitive framework:** “Cue → Action → Example” pattern to reduce ambiguity.

## Option B — Frontmatter Guidance Map
**Format:** Add frontmatter keys per section: `why`, `how`, `example`. Body stays ultra‑clean with inline examples only.
**Why:** Keeps the visual card short while still storing guidance in one place.
**Cognitive framework:** Separation of “instruction” vs “execution” reduces clutter during use.

## Option C — Tip Callout per Section
**Format:** Small callout under each header with “Why this matters + Example.” Inline examples stay.
**Why:** Adds just‑in‑time context; good for learning stage.
**Cognitive framework:** Just‑in‑time explanation improves retention without long paragraphs.

## Option D — Compact Top Table
**Format:** Single table at top with Section | Why | Example. Sections below contain only inline examples.
**Why:** One‑glance guidance, then pure execution below.
**Cognitive framework:** Chunking: guidance block first, action block second.

## Differences Summary
- **A** = smallest footprint; fastest to fill.
- **B** = cleanest body; guidance lives in frontmatter.
- **C** = most instructional; slightly longer.
- **D** = most structured; guidance centralized at top.

## Recommendation (Pareto)
Start with **A** or **B** for lowest load; shift to **C** if onboarding/teaching is the priority.
